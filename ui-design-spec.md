# Solonest UI 设计规范文档

## 一、品牌设计

### 1.1 产品信息
- **产品名称**：Solonest
- **产品类型**：微信小程序
- **品牌标语**：Connect, share, thrive
- **设计关键词**：warm, hopeful, happy

### 1.2 色彩系统
- **主色（Primary）**：#FFB347（温暖橙）- 代表温暖和能量，营造充满希望和快乐的环境
- **辅色（Secondary）**：#81D4FA（信任蓝）- 增添可信度和开放性，创造欢迎的连接空间
- **强调色（Accent）**：#CE93D8（创意紫）- 注入创造力和想象力，提升用户体验
- **背景色**：#FFFFFF（白色）、#F5F5F5（浅灰）
- **文字色**：
  - 主文字：#333333
  - 次要文字：#666666
  - 辅助文字：#999999
- **功能色**：
  - 成功：#4CAF50
  - 警告：#FF9800
  - 错误：#F44336
  - 链接：#81D4FA

### 1.3 字体系统
- **主字体（标题/品牌）**：Comfortaa - 圆润友好的字体，体现温暖和亲和力
- **正文字体**：Open Sans - 确保优秀的可读性和现代感
- **中文字体**：微信默认字体（-apple-system, BlinkMacSystemFont, "PingFang SC"）

### 1.4 字号规范
- **大标题**：32px / Bold
- **标题**：24px / Bold
- **副标题**：20px / Medium
- **正文**：16px / Regular
- **辅助文字**：14px / Regular
- **小字**：12px / Regular

## 三、核心页面设计规范

### 3.1 发现页（推荐用户列表）

#### 3.1.1 页面布局
- **顶部**：搜索框（占位符："搜索用户任何信息"）
- **主体**：用户卡片列表（一行一个）
- **加载**：下拉刷新 + 上滑加载更多

#### 3.1.2 用户卡片展示内容
- 头像（左侧，圆形，60px）
- 昵称（粗体，16px）
- 个人标签（标签样式，可横向滚动）
- 我能提供的（14px，灰色，最多显示2行）
- 位置（图标 + 城市名称）
- 在线状态（绿点 = 在线，灰色文字 = "X分钟前在线"）

#### 3.1.3 搜索功能
- **搜索历史**：显示最近2条
- **搜索结果**：仅显示用户（与推荐列表样式一致）
- **空状态**：插画 + "未找到相关结果\n试试其他关键词"

---

### 3.2 用户详情页（个人主页）

#### 3.2.1 页面结构（从上到下）

**头部区域**：
- 背景图（渐变色，使用品牌色）
- 头像（居中，80px，圆形）
- 昵称（居中，20px，粗体）
- 位置 + 在线状态（居中，14px）

**个人标签区域**：
- 标签横向排列，可滚动
- 标签样式：圆角矩形，品牌色边框，点击可查看同类用户

**关注数据区域**：
- 关注数 | 受关注数（横向排列，可点击）
- 使用数字强调样式（数字用品牌橙色 #FFB347，24px，粗体）
- 中间用竖线分隔

---

**个人信息区域**（上半部分）：

1. **爱好**
   - 标题："爱好"（16px，粗体，左对齐）
   - 使用彩色图标展示（48px × 48px，圆角 16px）
   - 横向排列，最多显示 6 个
   - 上下间距：16px

2. **技能**
   - 标题："技能"（16px，粗体，左对齐）
   - 使用多彩标签展示（技能类标签样式）
   - 可换行显示
   - 上下间距：16px

3. **照片**
   - 标题："照片"（16px，粗体，左对齐）
   - 最多5张，网格展示（3列，圆角 12px）
   - 间距 8px
   - 上下间距：16px

**分隔线**（1px，#E0E0E0，上下间距 24px）

---

**商业信息区域**（下半部分）：

1. **我能提供的**（必填）
   - 标题："我能提供的"（16px，粗体，左对齐）
   - 每行一条，最多10行，每行最多50字
   - AI 生成的标签展示在文字下方
   - 上下间距：16px

2. **我正在做**（必填）
   - 标题："我正在做"（16px，粗体，左对齐）
   - 每行一条，最多5行，每行最多50字
   - 描述当前正在进行的项目/业务
   - 发布需求时必须关联其中一条
   - 上下间距：16px

3. **合作方式偏好**（必填）
   - 标题："合作方式偏好"（16px，粗体，左对齐）
   - 文字描述，最多100字
   - 上下间距：16px

4. **工作经验**
   - 标题："工作经验"（16px，粗体，左对齐）
   - 自由文本，最多500字
   - AI 生成的标签展示在文字下方
   - 上下间距：16px

5. **补充资料**
   - 标题："补充资料"（16px，粗体，左对齐）
   - 最多3张图片，网格展示（3列，圆角 12px）
   - 上下间距：16px

**分隔线**（1px，#E0E0E0，上下间距 24px）

---

**评价区域**：
- 标题："合作评价"（16px，粗体，左对齐）
- 默认显示前3-5条评价
- 点击"查看更多"展开全部
- 评价卡片内容：
  - 评价者头像 + 昵称
  - 总评分（10分制，大字号显示）
  - 三个维度分数（沟通风格、资源真实性、履约情况）
  - 文字评价（最多显示2行，可展开）
  - 评价时间
- 上下间距：16px

**底部操作区**：
- 发起聊天按钮（主按钮，品牌主色）
- 关注/取消关注按钮（次按钮）
- 举报按钮（文字按钮，右上角）
- 评价TA按钮（仅当沟通超过10句时显示）

## 五、交互设计规范

### 5.1 注册/登录流程
1. **登录页**：
   - 手机号输入框
   - 测试期：直接登录按钮（无需验证码）
   - 正式版：获取验证码 + 验证码输入框
2. **首次注册**：
   - 注册成功后直接进入"发现"页
   - 不强制填写资料
3. **资料完善提醒**：
   - 未填写必填项（我能提供的、我正在做、合作方式偏好）时，系统不展示该用户
   - 点击聊天按钮时，弹窗提示"请先完善必填信息"
   - 点击发布需求按钮时，如未填写"我正在做"，弹窗提示"请先完善'我正在做'信息"

### 5.2 AI 标签生成
- **触发时机**：用户保存"我能提供的"或"工作经验"后
- **生成方式**：后台自动生成
- **展示位置**：在原始文字描述下方
- **用户操作**：可以编辑/删除标签
- **标签样式**：与个人标签一致

### 5.3 需求响应流程
1. 用户在需求列表点击需求卡片
2. 进入需求详情页
3. 点击"感兴趣"按钮
4. 系统发送通知给需求发布者
5. 发布者收到通知："XXX 对你的需求感兴趣"
6. 发布者可以主动联系对方

### 5.4 评价流程
1. **评价入口**：对方个人主页的"评价TA"按钮
2. **评价权限**：仅当双方沟通超过10句时显示按钮
3. **评价页面**：
   - 三个维度评分（1-10分）：
     - 沟通风格（权重 0.8）
     - 资源真实性（权重 0.8）
     - 履约情况（权重 1.4）
   - 文字评价（最多200字）
   - 提交按钮
4. **总评分计算**：(沟通风格×0.8 + 资源真实性×0.8 + 履约情况×1.4) / 3
5. **评价展示**：立即显示在对方主页，按时间倒序

### 5.5 举报流程
1. **举报入口**：个人主页、聊天窗口的举报按钮
2. **举报页面**：
   - 选择举报类型：
     - 垃圾广告
     - 虚假资源
     - 恶意骚扰
     - 违规内容
     - 其他（需填写说明）
   - 上传截图/证据（可选，最多3张）
   - 提交按钮
3. **举报反馈**：处理完成后通过系统通知告知结果

### 5.6 关注功能
- **关注按钮**：个人主页底部操作区
- **关注效果**：
  - 直接关注，无需对方同意
  - 关注的用户发布新需求时收到通知
  - 可在"我的关注"列表查看
- **取消关注**：再次点击按钮即可取消

## 六、状态与反馈设计

### 6.1 加载状态
- **Loading 动画**：转圈图标，品牌主色
- **位置**：页面中央
- **文字**：可选显示"加载中..."

### 6.2 空状态
所有空状态均包含：简洁插画 + 提示文案

1. **发现页无推荐**：
   - "还没有找到合适的伙伴\n完善你的资料，获得更精准推荐"
2. **需求广场无需求**：
   - "暂无合作需求\n发布你的第一个需求吧"
3. **消息列表无聊天**：
   - "还没有消息\n去发现页找到感兴趣的伙伴开始聊天吧"
4. **搜索无结果**：
   - "未找到相关结果\n试试其他关键词"

### 6.3 错误提示
**轻度错误**（网络请求失败、输入格式错误）：
- Toast 提示，3秒自动消失
- 示例："网络连接失败，请稍后重试"

**中度错误**（操作失败、权限不足）：
- Toast 提示 + 引导按钮
- 示例："请先完善必填信息" + "去完善"按钮

**重度错误**（账号被封禁、严重异常）：
- 弹窗提示，需要点击确认
- 示例："你的账号已被封禁，原因：XXX"

### 6.4 成功反馈
- Toast 提示，2秒自动消失
- 示例："发布成功"、"保存成功"、"关注成功"

## 七、数据展示规范

### 7.1 时间显示格式
- **1分钟内**：刚刚
- **1小时内**：X分钟前
- **24小时内**：X小时前
- **7天内**：X天前
- **超过7天**：MM-DD
- **超过1年**：YYYY-MM-DD

### 7.2 长文本截断
- **需求卡片**：最多3行，超出显示"...展开"
- **评价文字**：最多2行，超出显示"...展开"
- **我能提供的**（卡片中）：最多2行，超出显示"...展开"

### 7.3 图片处理
- **上传限制**：最大2MB，自动压缩
- **支持格式**：JPG、PNG、WEBP
- **展示方式**：点击放大，支持左右滑动切换

## 十、位置信息

### 10.1 位置获取
- 优先自动获取微信授权定位
- 用户可手动修改城市

### 10.2 位置展示
- **精度**：精确到城市（如"北京"）
- **个人主页**：必须是具体城市
- **需求发布**：可选择"远程"

### 10.3 在线状态
- **在线判断**：打开小程序为在线，切换到后台5分钟内仍显示在线
- **状态展示**：在线 / 离线 + 最后在线时间
- **隐私设置**：不可隐藏在线状态

## 十一、特殊规则

### 11.1 用户展示规则
- 未填写必填项（我能提供的、我正在做、合作方式偏好）的用户：
  - 不在发现页展示
  - 不在搜索结果中出现
  - 无法发起聊天
  - 无法发布需求

### 11.2 聊天限制
- 仅支持文字消息和 Emoji
- 聊天记录仅保留1个月
- 超过1个月的会话自动从列表中移除

### 11.3 评价权限
- 仅当双方沟通超过10句时，才显示"评价TA"按钮
- 单方评价，无需双方互评
- 评价后立即显示在对方主页
