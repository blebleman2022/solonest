个人 / 一人公司业务对接社交产品 PRD
一、产品概述
1.1 产品定位
聚焦电商、技术服务、新媒体等领域的个人 / 一人公司用户，提供无门槛的业务对接与资源链接平台，核心满足临时、灵活的合作需求（如项目分成、计件、志愿者等形式），通过 “资源展示 + 人格化呈现 + 高效匹配”，解决合作需求不稳定、回报不可预期的痛点，区别于求职类、技能售卖类社交产品。
1.2 目标用户
核心人群：电商从业者、技术服务提供者、新媒体运营者等个人 / 一人公司形态用户（无明确成立时长限制）
用户需求：需求方（临时业务需求无法招聘长期员工）、提供方（拒绝长期稳定形态，偏好灵活合作方式）
二、核心功能模块
2.1 个人主页模块
2.1.1 基础信息
必填项：我能提供的、我正在做、合作方式偏好
选填项：个人爱好、技能、照片、工作经验
补充材料：支持上传合作案例、资质证明等（无需平台审核，最多3张图片）
2.1.2 我能提供的展示规则
用户可按 “一行一条” 形式输入资源描述（每行最多50字，最多10行）
系统通过 AI 提取输入文字中的核心信息，自动生成标签（如输入 “电商供应链对接”，AI 提取标签 “电商供应链”）
标签与原始文字描述在个人主页同时展示（标签可点击查看同类资源用户）
用户可编辑或删除 AI 生成的标签
2.1.3 我正在做展示规则
用户可按 "一行一条" 形式输入当前正在进行的项目/业务（每行最多50字，最多5行）
此字段为必填项，用于发布需求时关联具体项目
展示在个人主页"商业"Tab 中，位于"我能提供的"之后
2.2 需求发布模块
2.2.1 发布规则
需求形式：用户可发布文字描述的合作需求（最多500字）
关联项目：发布需求时必须从"我正在做"列表中选择一条进行关联
工作地点：可选择具体城市或"远程"
操作权限：支持编辑、删除已发布需求
曝光机制：暂不支持付费置顶，仅默认自然曝光
发布限制：未填写"我正在做"的用户无法发布需求
2.3 搜索功能模块
搜索逻辑：基于关键字 / 组合关键字搜索，覆盖用户个人主页所有信息（我能提供的、我正在做、合作方式、爱好、工作经验等）
搜索范围：在"发现"Tab 搜索用户，在"需求"Tab 搜索需求
搜索历史：显示最近2条搜索记录
筛选方式：暂不设置细分字段筛选，仅通过关键字组合实现精准搜索
2.4 系统推荐模块
匹配逻辑：完全依赖系统默认算法，无需用户自定义筛选条件
推荐内容：基于用户个人主页信息（行业、资源、合作方式等）匹配潜在合作对象
展示形式：推荐列表展示用户核心信息（头像、行业、核心资源标签、合作方式）
2.5 沟通系统模块
沟通门槛：用户需完善必填信息后才可发起私聊
沟通功能：仅支持文字消息和 Emoji 表情
聊天记录：仅保留1个月内的会话记录
沟通场景：初次合作沟通需在站内完成，后续合作对接由用户自主选择渠道
功能限制：无合作跟进功能（如进度提醒、文件共享、语音/视频通话等）
2.6 举报功能模块
举报入口：个人主页、聊天窗口均设置举报按钮
举报处理：用户发起举报后，平台后台审核，可执行账号临时封禁、违规内容删除操作
2.7 合作评价体系
评价触发：双方系统内沟通超过10句后，可在对方主页发起评价
评价维度：沟通风格（权重0.8）、资源真实性（权重0.8）、履约情况（权重1.4）
评分制度：10分制，总评分 = (沟通风格×0.8 + 资源真实性×0.8 + 履约情况×1.4) / 3
评价方式：单方评价，无需双方互评
评价展示：评价结果（总评分 + 三维度评分 + 文字评价）在用户个人主页公开展示，按时间倒序，默认显示前3-5条
文字评价：最多200字
2.8 关注功能模块
关注机制：用户可直接关注其他用户，无需对方同意
关注效果：
- 可在"我的关注"列表查看已关注用户
- 关注的用户发布新需求时收到系统通知
- 不影响推荐算法
关注数据：个人主页显示"关注数"和"受关注数"，可点击查看列表
三、信息展示规则
个人主页信息排序（从上到下）：
- 头部：背景图、头像、昵称、位置、在线状态
- 个人标签区域（从"我能提供的"AI 提取）
- 关注数据（关注数、受关注数）
- **个人信息区域**：
  - 爱好（彩色图标）
  - 技能（多彩标签）
  - 照片（网格展示）
- 分隔线
- **商业信息区域**：
  - 我能提供的
  - 我正在做
  - 合作方式偏好
  - 工作经验
  - 补充资料
- 分隔线
- 合作评价
需求列表展示：按发布时间倒序排列，每条需求显示关联的"我正在做"项目标签
需求详情展示：发布者信息 → 关联项目标签 → 需求内容 → 工作地点 → 发布时间
搜索结果展示：按匹配度排序，优先展示包含核心关键字的用户 / 需求
四、违规处理机制
违规类型：垃圾广告、虚假资源、恶意骚扰、违规内容发布等
处理方式：
轻度违规：删除违规内容，账号临时封禁（1-3 天）
重度违规：账号永久封禁，清除所有发布内容及个人信息
反馈机制：用户举报后，后台在 72 小时内处理并通过系统消息反馈处理结果
五、非功能需求
产品形态：微信小程序
性能要求：页面加载时间≤2 秒，搜索响应时间≤1 秒，消息发送 / 接收延迟≤0.5 秒
兼容性要求：支持微信小程序最新版本，兼容 iOS 和 Android 系统
安全要求：用户聊天数据加密存储，个人信息（手机号）脱敏展示
AI 标签提取：
- 准确率≥85%
- 支持从"我能提供的"和"工作经验"中提取关键词
- 支持常见行业资源关键词提取（如电商、技术服务、新媒体领域核心词汇）
- 用户可编辑或删除生成的标签
位置服务：
- 优先使用微信授权定位自动获取城市
- 用户可手动修改城市
- 精确到城市级别
在线状态：
- 打开小程序为在线，切换到后台5分钟内仍显示在线
- 显示"最后在线时间"
- 不可隐藏在线状态
六、产品边界
不提供合作履约担保、资金托管等金融相关功能
不支持线下活动组织、跨地域限制等附加功能
暂不开放付费推广、会员体系等商业化功能
不介入用户合作纠纷处理，仅提供评价体系供用户参考
聊天功能仅支持文字和 Emoji，不支持图片、文件、语音、视频等富媒体内容

七、字段规范
| 字段 | 规则 |
|------|------|
| 昵称 | 2-20个字符 |
| 我能提供的 | 每行最多50字，最多10行，必填 |
| 我正在做 | 每行最多50字，最多5行，必填 |
| 合作方式偏好 | 最多100字，必填 |
| 工作经验 | 最多500字，选填 |
| 个人爱好 | 最多6个图标，选填 |
| 个人技能 | AI自动生成标签，选填 |
| 个人照片 | 最多5张，选填 |
| 补充资料 | 最多3张图片，选填 |
| 需求内容 | 最多500字 |
| 评价文字 | 最多200字 |

八、通知系统
通知类型：
1. 新消息通知
2. 有人对我的需求感兴趣
3. 有人关注我
4. 收到新评价
5. 系统公告/更新通知
6. 举报处理结果通知
7. 账号异常提醒

通知方式：仅站内通知（微信小程序无 App 推送）
通知展示：在"我的"页面顶部显示通知图标 + 未读数量
